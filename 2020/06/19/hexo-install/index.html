个人搭建HEXO博客系统的过程
发表于 2020-06-19| 更新于 2020-06-19|前端
阅读量:

背景

这两天抽空搭建了个人的博客，使用的hexo博客系统+butterfly主题包搭建。下面回顾下整个过程，对重点内容做下备忘。首先放上参考链接：
hexo文档：https://hexo.io/zh-cn/docs/
butterfly主题文档：https://demo.jerryc.me/

hexo搭建

安装前提

安装 Hexo 前，需要先安装下列应用程序：

  • Node.js (Node.js 版本需不低于 8.10，建议使用 Node.js 10.0 及以上版本)
  • Git

安装hexo

以上应用程序安装完成后，即可使用 npm 安装 Hexo。
$ npm install -g hexo-cli
输入$ hexo version看是否安装成功。

建站

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹folder中新建站点。

新建初始化完成后，指定文件夹folder下的目录内容如下：

_config.yml
网站的配置 信息，您可以在此配置网站个性化自定义的参数和信息，每个主题下也有一个_config.yml文件，一般是主题在hexo基础上拓展的配置信息。另外在主题的languages文件夹下的*.yml文件是对应语言下界面元素的本地化信息。
scaffolds
模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。
Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。
source
资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。每个主题下也有一个source文件夹，用于存放除md文件之外的静态资源。静态资源引用时以source文件夹为网站根目录。

安装主题

将下载好的butterfly主题包解压放到theme文件夹下即可。
修改站点配置文件_config.yml，把主题改为butterfly。

前往你的 Hexo 博客的根目录，输入

修改生成md文件的title和type，另外link是在hexo根目录下创建source/_data文件夹，创建一個文件link.yml

开启评论系统

搜索主题文件夹内的配置文件_config.yml，查找comments，可定位到valine等评论系统的设置，通过修改配置可开启评论系统。另外通过Front-matter的comments设置禁用tags等页面的评论功能。

至此博客站点已经完成搭建，只要按个人具体要求进一步修改配置参数和信息，一个完全个性化的博客就诞生了。

hexo命令

init
$ hexo init [folder]
新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。
new
$ hexo new [layout] <title>
新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。
$ hexo new "post title with whitespace"
generate
$ hexo generate
生成静态文件。
该命令可以简写为$ hexo g
publish
$ hexo publish [layout] <filename>
发表草稿。
clean
$ hexo clean
清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。
server
$ hexo server
启动服务器。默认情况下，访问网址为： http://localhost:4000/
deploy
$ hexo deploy
部署网站。
该命令可以简写为：$ hexo d
list
$ hexo list <type>
列出网站资料。
version
$ hexo version
显示 Hexo 版本。

写作

你可以执行下列命令来创建一篇新文章或者新的页面。
$ hexo new [layout] <title>
您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

布局（Layout）
Hexo 有三种默认布局：post、page 和 draft。在创建者三种不同类型的文件时，它们将会被保存到不同的路径；而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。

Front-matter

Front-matter 是文件最上方以 — 分隔的区域，用于指定个别文件的变量，举例来说：

以下是预先定义的参数，您可在模板中使用这些参数值并加以利用。

分类和标签

只有文章支持分类和标签，您可以在 Front-matter 中设置。在其他系统中，分类和标签听起来很接近，但是在 Hexo 中两者有着明显的差别：分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次。

标签插件（Tag Plugins）

标签插件和 Front-matter 中的标签不同，它们是用于在文章中快速插入特定内容的插件。详见https://hexo.io/zh-cn/docs/tag-plugins 文档说明。

部署更新

安装 hexo-deployer-git。
$ npm install hexo-deployer-git --save
修改_config.yml中的deploy配置。

生成站点文件并推送至远程库，执行 $ hexo clean && hexo deploy。
每次提交自定义message，可使用$ hexo deploy --message "info"。

结语

通过本次搭建熟悉了hexo博客系统，也对butterfly主题包的扩展功能有所了解，掌握了markdown的基本语法。
